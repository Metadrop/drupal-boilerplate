# Instrucciones para conectar proyecto con entornos remotos

## Aliases de drush
 Editar drush/sites/metadrop.site.yml  y poner los datos necesarios.

 Seguramente cambiar nombre del fichero para que tenga otro alias.


## Configurar drush

Editar `drush/drush.yml` para ajustarlo, entre otras cosas cambiar el password por defecto.

## Permnitar acceder a entornos remotos desde el docker

Descomentar las líneas en docker-compose override:

```
  php:
    environment:
      SSH_AUTH_SOCK: /ssh-agent
```

y

```
    volumes:
      - $SSH_AUTH_SOCK:/ssh-agent
```
